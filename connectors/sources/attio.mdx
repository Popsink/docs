The Attio Source Connector integrates with Attio, a modern Customer Relationship Management (CRM) platform designed for data-driven teams. Attio offers a flexible and customizable approach to manage customer data, interactions, and relationships. It allows users to create custom objects, fields, and workflows to match their specific business processes.

The connector enables real-time synchronization of data from Attio to your data pipeline, facilitating analytics, reporting, and integration with other business systems. This connector is particularly useful for companies looking to leverage their CRM data for advanced analytics, customer insights, and cross-functional collaboration.

## Prerequisites

<ParamField path="Attio Account" type="requirement" required>
  An active Attio account with connection privileges to access and replicate data
</ParamField>

## Connecting Attio

<Steps>
  <Step title="Select Attio Connector">
    In the source selection menu, select the Attio connector and click the **Connect** button.
    
    ![Attio Connector Selection](/images/attio-select.png)
  </Step>
  
  <Step title="Authenticate">
    A new window will open, prompting you to log in to your Attio account. After logging in, grant the necessary permissions to allow Popsink to access your Attio data.
    
    ![Attio OAuth Authorization](/images/attio-oauth.png)
  </Step>
  
  <Step title="Select Objects">
    Once authenticated, you can:
    - Give your connector a name
    - Select the Attio objects you wish to track
    
    In the "Table selection", you'll see a list of available objects in your Attio CRM. Select the objects you wish to replicate to Popsink.
    
    ![Attio Object Selection](/images/attio-objects.png)
  </Step>
  
  <Step title="Create and Sync">
    Once selected, click **Create connector**. Your connector is ready! You can now use the "Subscribe to all" button to sync all objects from the connector with the destination of your choice.
    
    ![Attio Connector Ready](/images/attio-ready.png)
  </Step>
</Steps>

## Configuration

### OAuth Authentication

<Note>
Attio uses OAuth 2.0 for authentication. No manual credential entry is required - simply authorize Popsink through the OAuth flow.
</Note>

### Object Selection

<ParamField path="Objects" type="array" required>
  Select which Attio objects to replicate. Available objects include:
  - Standard objects (Companies, People, Deals)
  - Custom objects created in your Attio workspace
  - List objects
</ParamField>

<ParamField path="Sync Mode" type="select" default="incremental">
  - `full`: Complete data refresh
  - `incremental`: Only sync changes (recommended)
</ParamField>

## Best Practices

<Tip>
**Start Small:** Begin by replicating a few key objects, then expand as you validate the integration works for your use case.
</Tip>

<Tip>
**Monitor Custom Objects:** Attio's flexibility means your data model can evolve. Regularly review which custom objects are being replicated.
</Tip>

<Tip>
**Use Incremental Sync:** For most use cases, incremental sync provides the best balance of data freshness and API efficiency.
</Tip>

<Tip>
**Parent Records:** Attio supports parent-child relationships. Ensure you're replicating parent objects if you need child records.
</Tip>

## Troubleshooting

<AccordionGroup>
  <Accordion title="OAuth authorization failed" icon="circle-exclamation">
    - Ensure you're logging in with an Attio account that has the necessary permissions
    - Check that you've granted all required OAuth scopes
    - Try clearing browser cookies and authorizing again
  </Accordion>
  
  <Accordion title="Object not appearing in list" icon="question">
    - Verify the object exists in your Attio workspace
    - Check that you have permission to view the object
    - Refresh the object list in Popsink
  </Accordion>
  
  <Accordion title="Schema changes not detected" icon="rotate">
    The connector automatically handles schema evolution, but you may need to:
    - Wait for the next sync cycle
    - Manually trigger a schema refresh if available
    - Check connector logs for any schema detection errors
  </Accordion>
  
  <Accordion title="Slow initial load" icon="hourglass">
    For workspaces with large datasets:
    - Consider loading fewer objects initially
    - Monitor API rate limits
    - Contact support if issues persist
  </Accordion>
</AccordionGroup>
